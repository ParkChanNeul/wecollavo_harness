# Revision Policy

피드백 1회는 정해진 검토 시점에 고객이 한 번에 정리해 전달한 의견 묶음이다.

## Not Counted Against Feedback

- 단순 오탈자
- 링크 오류
- 파일 출력 오류
- 명백한 개발/표시 오류
- 계약 범위 안의 버그

## Counted As Feedback

- 디자인 방향 조정
- 버튼, 섹션, 레이아웃 변경
- 이미지 교체
- 문장 톤 변경
- 색상 변경
- 페이지 내 우선순위 변경

## Change Order

아래는 수정이 아니라 범위 변경 또는 새 작업이다.

- 새 페이지
- 새 자산
- 새 기능
- 새 컨셉
- 브랜드 방향 변경
- 타겟 변경
- 전체 컬러/모티브 변경
- 고객 제공 예정 자료를 WeCollavo가 새로 작성하는 경우

## Client-Safe Phrase

기본 피드백은 정해진 검토 시점에 한 번에 정리해 전달된 의견 묶음을 기준으로 산정합니다. 합의된 범위 안의 조정은 수정으로 처리되며, 새 페이지, 새 기능, 새 방향, 새 자산이 추가되는 경우 별도 범위로 산정됩니다.
