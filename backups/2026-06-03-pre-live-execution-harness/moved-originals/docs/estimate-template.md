## 1. 문서 목적과 사용 경계

이 문서는 모든 고객에게 같은 견적을 보내기 위한 문안 템플릿이 아니다.

Track 1 고객의 상황을 확인하고, 합의된 핵심 자산 1개를 기준으로 견적을
산정하기 위한 내부 가이드다. 고객별 확정 가격, 특수 사정, 미팅에서 나온
판단은 `project/<client>/` 또는 `harness/workspaces/<client>/`에 기록한다.

이 문서에는 고객별 적용값을 역류시키지 않는다.

## 2. 견적 전 확인해야 할 입력값

견적을 작성하기 전에 아래 값을 먼저 채운다.

```text
고객명:
작업명:
핵심 자산:
사용처:
자료 상태:
페이지 수:
홈페이지 범위:
로고 작업 여부:
수정 횟수:
자료 제공 마감일:
예상 일정:
예산 감각:
최종 의사결정자:
```

필수 판단:

- 고객이 가장 먼저 외부에 보여줘야 하는 자산은 무엇인가
- 현재 자료가 외부 공개 가능한 수준인가
- 고객이 자료를 정리해서 제공할 수 있는가
- WeCollavo가 콘텐츠를 직접 정리하거나 작성해야 하는가
- 일정 압박이 있는가
- 기본 수정 범위를 넘어설 가능성이 있는가

## 3. 고객 상황 분류

고객 상황을 먼저 분류한 뒤 견적 구조를 잡는다.

| 상황 | 판단 기준 | 견적 영향 |
| --- | --- | --- |
| 기존 자산 마감 | 이미 자료가 있고 외부 공개 수준으로 정리해야 함 | 진단, 구조 정리, 핵심 자산 마감 중심 |
| 자료 정리 병목 | 자료는 있으나 정리/선별/문장화가 안 됨 | 콘텐츠 정리와 작성 범위가 커짐 |
| 홈페이지 마감 | 기존 구조가 있고 실제 콘텐츠 반영이 필요함 | 문구, 이미지, 섹션, 반응형 확인 범위 산정 |
| 회사소개서 제작 | 페이지 수와 난이도 산정이 필요함 | 페이지 단가와 자료 완성도 기준 산정 |
| 로고/CI 정리 | 기존 로고 파일 정리, 벡터화, 리터칭이 필요함 | 기본 범위 포함 여부와 별도 옵션 구분 |
| 범위 확장 위험 | 고객이 여러 자산을 한 번에 요구함 | 핵심 자산 1개와 추가 옵션을 분리 |

## 4. 핵심 자산 선택

Track 1의 기본 흐름은 합의된 핵심 자산 1개를 제작하거나 마감하는 것이다.

가능한 핵심 자산:

- 브로셔
- 회사소개서
- 홈페이지 수정/마감

로고 파일 정리, 벡터화, 리터칭은 핵심 자산을 보조하는 옵션으로 볼 수 있다.
신규 로고 디자인은 기본 포함이 아니라 별도 옵션이다.

선택 질문:

1. 지금 가장 급한 외부 공개 자산은 무엇인가?
2. 이 자산은 누구에게 보여주는가?
3. 이 자산이 완성되지 않으면 어떤 영업/제휴/투자 기회가 막히는가?
4. 고객이 제공할 수 있는 원본 자료는 어디까지인가?
5. WeCollavo가 직접 정리해야 하는 범위는 어디부터인가?

## 5. 포함 범위

기본 포함 범위는 고객별 핵심 자산에 맞춰 조정한다.

- 브랜드 자산 진단
- 기존 자료 검토
- 외부 공개가 어려운 지점 정리
- 핵심 메시지와 구조 정리
- 합의된 핵심 자산 1개 제작 또는 마감 방향 설정
- Media Funnel Map
- Next 30 Days Operating Plan
- Proof Loop 기준 설정

핵심 자산이 홈페이지인 경우:

- 기존 구조 검토
- 우선 수정 페이지 확인
- 문구, 이미지, 섹션 교체 범위 산정
- 문의 흐름과 CTA 확인

핵심 자산이 브로셔 또는 회사소개서인 경우:

- 페이지 수 산정
- 페이지별 난이도 구분
- 자료 제공 상태 확인
- 문장 흐름과 섹션 구조 정리

## 6. 제외 범위

아래 항목은 기본 견적에 포함하지 않는다.

- 무제한 수정
- 전체 리브랜딩
- 신규 로고 디자인
- 대형 홈페이지 전체 리빌드
- 사진/영상 촬영
- 대량 카피라이팅
- 상세 제품 설명 전체 작성
- 광고 집행
- 매출 보장
- 지속적인 채널 운영과 게시 대행
- 법무, 입찰, 인증 문구의 최종 검증

## 7. 가격 참고 기준

아래 금액은 참고 기준이다. 최종 견적은 핵심 자산, 페이지 수, 자료 상태,
콘텐츠 작성 범위, 홈페이지 구조 변경 여부, 로고/CI 작업 여부, 수정 횟수,
일정 압박에 따라 조정한다.

## 7-1. 현장 가격 안전장치

미팅 중 가격은 확정 견적이 아니라 안전한 가격 프레임으로 말한다.

```text
현장 진단 제안
자료 검토 전 기준

-> 1차 제안서
자료 일부 확인 후 기준

-> 최종 견적서
전체 자료 확인 후 확정
```

고객에게는 아래 문장을 기본으로 사용한다.

> 오늘은 방향과 예상 범위를 먼저 잡아드릴게요. 다만 정확한 견적은 기존
> 자료를 확인한 뒤 최종 확정하는 게 서로 안전합니다.

### 기본 착수비 참고 기준

기본 착수비 참고 범위: **120만~200만 원**

포함 가능 항목:

- 브랜드 자산 진단
- 기존 자료 검토
- 핵심 메시지와 구조 정리
- 핵심 자산 1개 제작 또는 마감 방향 설정
- Media Funnel Map
- Next 30 Days Operating Plan
- Proof Loop 기준 설정

### 브로셔/회사소개서 제작비 참고 기준

| 구분 | 기준 | 참고 단가 |
| --- | --- | ---: |
| 일반 페이지 | 텍스트와 이미지 중심 구성 | 8만 원 / p |
| 복잡 페이지 | 도표, 사업 구조, 제품군 정리 포함 | 12만~15만 원 / p |
| 고난도 페이지 | 기술 설명, 투자/제휴 설득 구조 포함 | 별도 협의 |

### 홈페이지 수정/마감 참고 기준

| 구분 | 기준 | 참고 금액 |
| --- | --- | ---: |
| 경량 수정 | 문구, 이미지, 일부 섹션 수정 | 30만~70만 원 |
| 일반 마감 | 주요 페이지 콘텐츠 추가/수정 | 80만~150만 원 |
| 구조 개편 | 페이지 구조, 흐름, CTA 재설계 | 별도 협의 |

### 로고 관련 작업 참고 기준

| 구분 | 기준 | 산정 방식 |
| --- | --- | --- |
| 기존 로고 파일 정리 | 파일 포맷 정리, 간단 적용 | 범위별 산정 |
| 로고 벡터화 | 인쇄/웹 사용 가능한 벡터 파일 제작 | 범위별 산정 |
| 로고 리터칭 | 기존 로고를 크게 바꾸지 않는 개선 | 별도 협의 |
| 신규 로고 디자인 | 새 로고 기획 및 디자인 | 별도 옵션 |

## 8. 추가비 발생 조건

아래 조건이 발생하면 별도 비용 또는 일정 조정이 필요하다.

| 조건 | 이유 | 처리 기준 |
| --- | --- | --- |
| 페이지 수 증가 | 디자인, 문장 정리, 검토 시간이 늘어남 | 페이지 단가 기준 재산정 |
| 새로운 자산 추가 | 핵심 자산 1개 범위를 벗어남 | 별도 옵션 견적 |
| 자료 미정리 | WeCollavo가 선별, 정리, 작성까지 해야 함 | 콘텐츠 정리 비용 추가 |
| 대량 문구 재작성 | 단순 수정이 아니라 카피라이팅 업무로 전환 | 별도 협의 |
| 구조 개편 | 기존 구조 보완을 넘어 흐름 재설계가 필요함 | 별도 협의 |
| 최종 승인 후 변경 | 승인 이후 새 작업이 발생함 | 별도 일정/비용 산정 |

## 9. 자료 제공 기준

원활한 진행을 위해 고객이 제공해야 할 자료를 먼저 확정한다.

기본 자료:

- 기존 브로셔 또는 회사소개서 원본 파일
- 홈페이지 관리자 접근 권한 또는 수정 가능한 원본
- 로고 파일
- 회사 소개 문구
- 사업 영역별 설명 자료
- 제품/서비스 이미지
- 레퍼런스 또는 기존 영업 자료
- 다음 미팅/제안/영업 일정

자료 제공 기준:

- 고객 자료가 정리되어 있으면 제작비와 일정이 줄어들 수 있다.
- 자료 제공이 지연되면 납품 일정도 함께 조정된다.
- 공개 가능 여부가 불분명한 자료는 고객 확인 후 사용한다.

## 10. 일정 산정 기준

기본 일정 참고 범위는 자료 수령 후 **10~14영업일**이다.

```text
Day 1-2: 자료 검토 및 방향 정리
Day 3-5: 구조/메시지 정리
Day 6-10: 디자인 및 콘텐츠 제작
Day 11-12: 1차 검토 및 수정
Day 13-14: 최종 마감 및 전달
```

일정은 아래 변수에 따라 조정한다.

- 페이지 수
- 홈페이지 범위
- 자료 제공 속도
- 고객 피드백 속도
- 콘텐츠 작성 필요량
- 외부 공개 전 검토 필요성

## 11. 수정 기준

기본 수정은 **2회**까지 포함하는 것을 기준으로 한다.

기본 수정에 포함될 수 있는 항목:

- 오탈자
- 문장 흐름 조정
- 이미지 교체
- 일부 섹션 조정

별도 협의가 필요한 항목:

- 새로운 페이지 추가
- 새로운 사업부 또는 서비스 추가
- 대량 문구 재작성
- 최종 승인 후 방향 변경
- 대표님 또는 담당자 피드백이 여러 경로로 분산되는 경우

피드백은 정리된 문서로 전달받는 것을 기준으로 한다.

## 12. Proof Loop 기준

Proof Loop는 매출 보장이 아니다.

납품 후 30일 안에 브랜드 자산이 실제로 쓰였는지, 고객사의 외부 설명력이
개선됐는지, 다음 실행으로 이어졌는지 확인하기 위한 기준이다.

확인 신호:

- 완성 자료가 실제 영업/제휴/투자 미팅에서 사용됐는가
- 대표님 또는 담당자가 더 쉽게 설명할 수 있게 됐는가
- 홈페이지, 브로셔, 회사소개서를 외부에 공유했는가
- 추가 페이지, 홈페이지 확장, 월간 운영 요청이 생겼는가

## 13. 고객 전달 문구 조립 블록

아래 블록을 고객 상황에 맞게 조합한다. 그대로 복사하기보다 고객별 입력값과
미팅 답변을 반영해 수정한다.

### 문제 진단 블록

```text
이번 작업은 단순히 디자인을 고치는 일이 아니라, 기존 자료를 외부에서 이해하고
신뢰할 수 있는 브랜드 자산으로 정리하는 작업입니다.
```

### 핵심 자산 제안 블록

```text
이번 단계에서는 모든 자산을 한 번에 확정하기보다, 먼저 가장 급한 외부 공개
자산 1개를 기준으로 범위와 일정을 잡는 것이 적합합니다.
```

### 가격 산정 설명 블록

```text
최종 견적은 핵심 자산, 페이지 수, 자료 제공 상태, 콘텐츠 작성 범위, 홈페이지
수정 범위, 로고 작업 여부에 따라 달라집니다. 먼저 이 기준을 정리한 뒤 실제
작업 범위에 맞춰 견적을 산정하겠습니다.
```

### 자료 제공/일정 조건 블록

```text
자료가 정리되어 전달되면 제작비와 일정은 줄어들 수 있습니다. 반대로 자료
정리와 문구 작성까지 WeCollavo가 맡아야 한다면 작업 범위와 일정이 함께
조정됩니다.
```

### 추가비/수정 기준 블록

```text
기본 수정 범위는 사전에 합의한 기준 안에서 진행하고, 새 페이지 추가, 대량
문구 재작성, 최종 승인 후 변경은 별도 일정과 비용으로 산정합니다.
```

### 다음 단계 블록

```text
다음 단계에서는 사용처, 핵심 자산, 필요한 자료, 수정 기준, 납품 일정을 먼저
확정한 뒤 견적서를 정리하겠습니다.
```
